# Security Policy

## Supported Versions

Use this section to tell people which versions receive security updates.

| Version | Supported |
| ------- | --------- |
| 5.1.x   | ✅         |
| 5.0.x   | ❌         |
| 4.0.x   | ✅         |
| < 4.0   | ❌         |

## Reporting a Vulnerability

Do not open public GitHub issues for security vulnerabilities. Preferred reporting channel:

- GitHub Security Advisories: https://github.com/klevze/phpColors/security/advisories (preferred)

If you cannot use GitHub Security Advisories, contact the maintainers privately by opening a confidential communication channel (for example, an encrypted email). If you require an encrypted option, reply with your preferred contact details and PGP fingerprint and we will add them to this document.

When reporting, please include:
- Affected version(s)
- A clear description of the issue and impact
- Reproduction steps or a proof-of-concept (if safe to share)
- Any exploit code or test cases (if available)
- Preferred disclosure and credit details

## Response & Timeline

We will:
- Acknowledge receipt within 72 hours
- Provide an initial triage and impact assessment within 7 days
- Prioritize fixing confirmed security issues and ship a patch for supported versions
- Provide regular updates until a fix and advisory are published

## Disclosure Policy

We prefer coordinated disclosure. We will work with the reporter to set a disclosure timeline, aiming to publish an advisory concurrently with or shortly after a fix is released.

## CVE Policy

We will request CVE identifiers for security issues that meet the CVE criteria. If you require a CVE during triage, please indicate that in your report.

## Severity & Backporting

Critical fixes will be backported to supported release lines where feasible. Supported versions are listed above.

## Acknowledgements

Reporters may request credit in the advisory; indicate your preference when reporting.
